**\#\# 3\. Neo4j 圖形設計**

**\#\#\# 3.1 節點設計**

**\#\#\#\# 節點標籤與屬性**

\`\`\`  
(:Station)  
  ├── 屬性:  
  │   ├── station\_id (string, unique) — 例："MS01", "NR03"  
  │   ├── name (string) — 例："Central Square"  
  │   ├── network (string) — "metro" 或 "national\_rail"  
  │   └── lines (array) — 例：\["M1", "M2"\]  
\`\`\`

**\*\*注意\*\***：所有站點都使用同一標籤 \`Station\`，但透過 \`network\` 屬性區分。

**\#\#\# 3.2 關係設計**

**\#\#\#\# 1\. CONNECTS\_TO 關係**  
\- **\*\*方向\*\***：單向 (\`from\_id\` → \`to\_id\`)  
\- **\*\*屬性\*\***：  
  \- \`line\` (string) — 路線編號，例 "M1", "NR1"  
  \- \`network\` (string) — "metro" 或 "national\_rail"  
  \- \`travel\_time\_min\` (integer) — 旅行時間（分鐘）  
\- **\*\*用途\*\***：表示網路內的直接連接

\`\`\`  
(Station {station\_id: "MS01"})  
  \-\[CONNECTS\_TO {line: "M1", travel\_time\_min: 3, network: "metro"}\]-\>  
(Station {station\_id: "MS05"})  
\`\`\`

**\#\#\#\# 2\. INTERCHANGE\_WITH 關係**  
\- **\*\*方向\*\***：雙向 (metro ↔ rail)  
\- **\*\*屬性\*\***：無（純連接）  
\- **\*\*用途\*\***：表示地鐵和國鐵之間的換乘

\`\`\`  
(Station {station\_id: "MS01"})  
  \-\[INTERCHANGE\_WITH\]-\>(Station {station\_id: "NR01"})  
(Station {station\_id: "NR01"})  
  \-\[INTERCHANGE\_WITH\]-\>(Station {station\_id: "MS01"})  
\`\`\`

**\#\#\# 3.3 索引與約束**

**\*\*必須在 Neo4j 中建立（以提高查詢效能）：\*\***

\`\`\`cypher  
\-- 唯一性約束（同時建立索引）  
CREATE CONSTRAINT station\_id\_unique  
FOR (s:Station)  
REQUIRE s.station\_id IS UNIQUE;

\-- 路線索引（用於篩選查詢）  
CREATE INDEX station\_network\_idx  
FOR (s:Station)  
ON (s.network);  
\`\`\`

**\#\#\# 4.2 第二階段：實現查詢函數**

在 \`databases/graph/queries.py\` 中實現 6 個查詢函數。每個函數對應一個特定的路由查詢用途。

**\#\#\#\# 函數 1：\`query\_shortest\_route()\` — 最快路線**

**\*\*目的\*\***：使用 Dijkstra 演算法找到時間最短的路線。

**\*\*輸入參數\*\***：  
\- \`origin\_id\` (str) — 起始站點ID，例："MS01"  
\- \`destination\_id\` (str) — 目的地站點ID，例："MS09"  
\- \`network\` (str) — 可選，"metro", "rail", 或 "auto"（預設自動推斷）

**\*\*輸出格式\*\***：  
\`\`\`python  
{  
    "found": True,  
    "origin\_id": "MS01",  
    "destination\_id": "MS09",  
    "total\_time\_min": 15,  
    "path": \[  
        {"station\_id": "MS01", "name": "Central Square"},  
        {"station\_id": "MS05", "name": "Westfield"},  
        {"station\_id": "MS09", "name": "Queensbridge"}  
    \],  
    "legs": \[  
        {"line": "M1", "travel\_time\_min": 3},  
        {"line": "M1", "travel\_time\_min": 12}  
    \]  
}  
\`\`\`

**\*\*Cypher 提示\*\***：  
\`\`\`cypher  
MATCH (o:Station {station\_id: $origin\_id}), (d:Station {station\_id: $destination\_id})  
CALL apoc.algo.dijkstra(o, d, 'CONNECTS\_TO', 'travel\_time\_min')  
YIELD path, weight  
RETURN  
  \[node in nodes(path) | {station\_id: node.station\_id, name: node.name}\] AS stations,  
  \[rel in relationships(path) | {line: rel.line, travel\_time\_min: rel.travel\_time\_min}\] AS legs,  
  weight AS total\_time\_min  
\`\`\`

**\*\*實現重點\*\***：  
\- 使用 APOC 的 \`dijkstra\` 演算法（已在 docker-compose.yml 中啟用）  
\- 處理 \`network\` 參數推斷（MS → metro, NR → rail）  
\- 返回完整的路徑和中間站點

**\---**

**\#\#\#\# 函數 2：\`query\_cheapest\_route()\` — 最便宜路線**

**\*\*目的\*\***：找到價格最低的路線（使用 travel\_time\_min 作為代理權重）。

**\*\*輸入參數\*\***：  
\- \`origin\_id\` (str)  
\- \`destination\_id\` (str)  
\- \`network\` (str) — "metro", "rail", 或 "auto"  
\- \`fare\_class\` (str) — "standard" 或 "first"（國鐵專用）

**\*\*輸出格式\*\***：  
\`\`\`python  
{  
    "found": True,  
    "total\_fare\_usd": 12.50,  
    "fare\_class": "standard",  
    "stations": \[...\],  
    "legs": \[...\]  
}  
\`\`\`

**\*\*實現提示\*\***：  
\- 使用 Dijkstra 找到路線（同 \`query\_shortest\_route\`）  
\- 呼叫 Relational 層的 \`query\_metro\_fare()\` 或 \`query\_national\_rail\_fare()\` 計算總價格  
\- 路線本身的邊上沒有直接的 fare 屬性，需從 Relational 資料庫查詢

**\*\*偽程式碼\*\***：  
\`\`\`python  
def query\_cheapest\_route(...):  
    \# 1\. 用 Dijkstra 找到路線（可能不是最短時間）  
    \# 2\. 計算中間站數  
    \# 3\. 根據 fare\_class 查詢國鐵或地鐵的票價  
    \# 4\. 返回路線和總票價  
\`\`\`

**\---**

**\#\#\#\# 函數 3：\`query\_alternative\_routes()\` — 替代路線（避開某站）**

**\*\*目的\*\***：找到多條路線，排除指定的中間站點（用於應對延誤或關閉）。

**\*\*輸入參數\*\***：  
\- \`origin\_id\` (str)  
\- \`destination\_id\` (str)  
\- \`avoid\_station\_id\` (str) — 要迴避的站點ID，例："NR03"  
\- \`network\` (str)  
\- \`max\_routes\` (int) — 最多返回路線數，預設 3

**\*\*輸出格式\*\***：  
\`\`\`python  
\[  
    {  
        "route": \[  
            {"station\_id": "NR01", "name": "Central Station"},  
            ...  
        \],  
        "total\_time\_min": 50  
    },  
    {  
        "route": \[...\],  
        "total\_time\_min": 65  
    }  
\]  
\`\`\`

**\*\*Cypher 提示\*\***：  
\`\`\`cypher  
MATCH p \= (o:Station {station\_id: $origin\_id})  
          \-\[:CONNECTS\_TO\*1..10\]-\>  
          (d:Station {station\_id: $destination\_id})  
WHERE NONE(n IN nodes(p) WHERE n.station\_id \= $avoid\_station\_id)  
RETURN \[n IN nodes(p) | {station\_id: n.station\_id, name: n.name}\] AS route,  
       reduce(t=0, r IN relationships(p) | t \+ r.travel\_time\_min) AS total\_time  
ORDER BY total\_time  
LIMIT $max\_routes  
\`\`\`

**\---**

**\#\#\#\# 函數 4：\`query\_interchange\_path()\` — 跨網路換乘路線**

**\*\*目的\*\***：找到跨越地鐵和國鐵邊界的路線（使用 INTERCHANGE\_WITH）。

**\*\*輸入參數\*\***：  
\- \`origin\_id\` (str) — 可為地鐵或國鐵站點  
\- \`destination\_id\` (str) — 另一個網路的站點

**\*\*輸出格式\*\***：  
\`\`\`python  
{  
    "found": True,  
    "stations": \[  
        {"station\_id": "MS01", "name": "Central Square"},  
        {"station\_id": "NR01", "name": "Central Station", "interchange": True},  
        ...  
    \],  
    "interchanges": \[  
        {"from": "MS01", "to": "NR01", "transfer\_time\_min": 3}  
    \],  
    "total\_time\_min": 25  
}  
\`\`\`

**\*\*Cypher 提示\*\***：  
\`\`\`cypher  
MATCH p \= (o:Station {station\_id: $origin\_id})  
          \-\[:CONNECTS\_TO|INTERCHANGE\_WITH\*1..20\]-\>  
          (d:Station {station\_id: $destination\_id})  
WHERE any(r IN relationships(p) WHERE type(r) \= 'INTERCHANGE\_WITH')  
RETURN nodes(p), relationships(p)  
ORDER BY length(nodes(p))  
LIMIT 1  
\`\`\`

**\*\*實現重點\*\***：  
\- 混合使用 \`CONNECTS\_TO\` 和 \`INTERCHANGE\_WITH\` 關係  
\- 標記換乘點  
\- 估算換乘時間（可固定值，例 3 分鐘）

**\---**

**\#\#\#\# 函數 5：\`query\_delay\_ripple()\` — 延誤波紋分析**

**\*\*目的\*\***：找出所有受某個延誤/關閉站點影響的其他站點（在指定跳數內）。

**\*\*輸入參數\*\***：  
\- \`delayed\_station\_id\` (str) — 延誤站點ID  
\- \`hops\` (int) — 搜尋距離（跳數），預設 2

**\*\*輸出格式\*\***：  
\`\`\`python  
\[  
    {  
        "station\_id": "MS02",  
        "name": "Riverside",  
        "hops\_away": 1,  
        "lines\_affected": \["M1"\]  
    },  
    {  
        "station\_id": "MS03",  
        "name": "Northgate",  
        "hops\_away": 1,  
        "lines\_affected": \["M1"\]  
    },  
    ...  
\]  
\`\`\`

**\*\*Cypher 提示\*\***：  
\`\`\`cypher  
MATCH (s:Station {station\_id: $station\_id})  
      \-\[r:CONNECTS\_TO\*1..$hops\]-\>(affected:Station)  
RETURN DISTINCT  
  affected.station\_id AS station\_id,  
  affected.name AS name,  
  min(length(r)) AS hops\_away,  
  affected.lines AS lines\_affected  
ORDER BY hops\_away  
\`\`\`

**\#\#\# 4.3 第三階段：實現細節與最佳實踐**

**\#\#\#\# 4.3.1 驅動程式管理 (Driver Management)**

**\*\*問題\*\***：當前 \`\_driver()\` 函數每次都建立新驅動，這在生產環境中效率低。

**\*\*解決方案\*\***：建立全局單例驅動程式。

**\*\*改進代碼\*\***（在 \`databases/graph/queries.py\` 開頭添加）：

\`\`\`python  
from neo4j import GraphDatabase  
from skeleton.config import NEO4J\_URI, NEO4J\_USER, NEO4J\_PASSWORD

\# 全局單例驅動程式（應用啟動時建立一次）  
\_DRIVER \= GraphDatabase.driver(  
    NEO4J\_URI,  
    auth\=(NEO4J\_USER, NEO4J\_PASSWORD),  
    max\_connection\_pool\_size\=50,  
)

def \_get\_driver():  
    """返回全局驅動程式。"""  
    return \_DRIVER

\# 在應用關閉時呼叫（例 Flask app context 或 Gradio shutdown）  
def close\_driver():  
    """關閉全局驅動程式。"""  
    \_DRIVER.close()  
\`\`\`

**\#\#\#\# 4.3.2 查詢模式標準化**

所有查詢函數應遵循此模式：

\`\`\`python  
def query\_xxx(...) \-\> dict | list:  
    """\[函數說明\]"""  
    with \_get\_driver().session() as session:  
        result \= session.run(  
            """  
            MATCH ...  
            RETURN ...  
            """,  
            param1\=param1\_value,  
            param2\=param2\_value,  
        )  
        \# 處理結果  
        if not result:  
            return {"found": False}  \# or return \[\]  
        \# 轉換為所需格式  
        return {...}  
\`\`\`

**\#\#\#\# 4.3.3 錯誤處理**

在每個查詢函數中添加基本錯誤捕捉：

\`\`\`python  
try:  
    with \_get\_driver().session() as session:  
        result \= session.run(...)  
        \# 處理結果  
except Exception as e:  
    print(f"Query error: {e}")  
    return {"found": False, "error": str(e)}  
\`\`\`

**\#\#\#\# 4.3.4 參數驗證**

在 \`query\_shortest\_route\` 等函數中，添加網路推斷邏輯：

\`\`\`python  
def \_infer\_network(station\_id: str) \-\> str:  
    """從站點ID推斷網路。"""  
    if station\_id.startswith("MS"):  
        return "metro"  
    elif station\_id.startswith("NR"):  
        return "national\_rail"  
    return "unknown"

def query\_shortest\_route(..., network: str \= "auto"):  
    if network \== "auto":  
        \# 推斷起點和終點的網路  
        origin\_network \= \_infer\_network(origin\_id)  
        dest\_network \= \_infer\_network(destination\_id)  
        \# 如果跨越網路，呼叫 query\_interchange\_path  
        if origin\_network \!= dest\_network:  
            return query\_interchange\_path(origin\_id, destination\_id)  
\`\`\`

**\#\#\# 7.1 避免的常見陷阱**

| 陷阱 | 原因 | 解決方案 |  
|------|------|--------|  
| 查詢返回多條路線時不排序 | 結果不確定 | 添加 \`ORDER BY\` 子句 |  
| 沒有處理 "not found" 情況 | 查詢返回空結果時崩潰 | 檢查 \`result.single()\` 是否為 None |  
| 混淆有向和無向關係 | CONNECTS\_TO 單向，但邊界查詢需雙向 | INTERCHANGE\_WITH 自動雙向，CONNECTS\_TO 需反向查詢 |  
| 忘記使用交通時間權重 | Dijkstra 結果不正確 | 確保傳遞 \`travel\_time\_min\` 到 apoc.algo.dijkstra |  
| 參數注入錯誤 | 安全性和語法問題 | 始終使用 \`$param\` 而非字符串拼接 |

**\#\#\#\#** **函數 1：\`query\_shortest\_route()\` — 最快路線**

**\*\*輸入\*\***：\`origin\_id\`, \`destination\_id\`, \`network="auto"\`

**\*\*輸出格式\*\***：  
\`\`\`python  
{  
   "found": True,  
   "origin\_id": "MS01",  
   "destination\_id": "MS09",  
   "total\_time\_min": 15,  
   "path": \[{"station\_id": "MS01", "name": "Central Square"}, ...\],  
   "legs": \[{"line": "M1", "travel\_time\_min": 3}, ...\]  
}  
\`\`\`

**\*\*Cypher\*\***：  
\`\`\`cypher  
MATCH (o:Station {station\_id: $origin\_id}), (d:Station {station\_id: $destination\_id})  
CALL apoc.algo.dijkstra(o, d, 'CONNECTS\_TO', 'travel\_time\_min')  
YIELD path, weight  
RETURN  
 \[node in nodes(path) | {station\_id: node.station\_id, name: node.name}\] AS stations,  
 \[rel in relationships(path) | {line: rel.line, travel\_time\_min: rel.travel\_time\_min}\] AS legs,  
 weight AS total\_time\_min  
\`\`\`

**\*\*實現重點\*\***：  
\- \`network="auto"\` 時用 \`\_infer\_network()\` 推斷  
\- 跨網路時自動轉呼叫 \`query\_interchange\_path()\`  
\- 使用 APOC \`dijkstra\`（已在 docker-compose 啟用）

**\---**

**\#\#\#\#** **函數 2：\`query\_station\_connections()\` — 站點直接連接**

**\*\*輸入\*\***：\`station\_id\`

**\*\*輸出格式\*\***：  
\`\`\`python  
\[  
   {"station\_id": "MS05", "name": "Westfield", "line": "M1", "travel\_time\_min": 3, "network": "metro"},  
   ...  
\]  
\`\`\`

**\*\*Cypher\*\***：  
\`\`\`cypher  
MATCH (s:Station {station\_id: $station\_id})-\[r:CONNECTS\_TO\]-\>(n:Station)  
RETURN n.station\_id AS station\_id, n.name AS name,  
      r.line AS line, r.travel\_time\_min AS travel\_time\_min, r.network AS network  
ORDER BY travel\_time\_min  
\`\`\`

**\---**

**\#\#\#\#** **函數 3：\`query\_delay\_ripple()\` — 延誤波紋分析**

**\*\*輸入\*\***：\`delayed\_station\_id\`, \`hops=2\`

**\*\*輸出格式\*\***：  
\`\`\`python  
\[  
   {"station\_id": "MS02", "name": "Riverside", "hops\_away": 1, "lines\_affected": \["M1"\]},  
   ...  
\]  
\`\`\`

**\*\*Cypher\*\***：  
\`\`\`cypher  
MATCH (s:Station {station\_id: $station\_id})-\[r:CONNECTS\_TO\*1..$hops\]-\>(affected:Station)  
RETURN DISTINCT  
 affected.station\_id AS station\_id,  
 affected.name AS name,  
 min(length(r)) AS hops\_away,  
 affected.lines AS lines\_affected  
ORDER BY hops\_away  
\`\`\`

**\---**

**\#\#\#\#** **函數 4：\`query\_alternative\_routes()\` — 替代路線（避開某站）**

**\*\*輸入\*\***：\`origin\_id\`, \`destination\_id\`, \`avoid\_station\_id\`, \`network="auto"\`, \`max\_routes=3\`

**\*\*輸出格式\*\***：  
\`\`\`python  
\[  
   {"route": \[{"station\_id": "NR01", "name": "Central Station"}, ...\], "total\_time\_min": 50},  
   ...  
\]  
\`\`\`

**\*\*Cypher\*\***：  
\`\`\`cypher  
MATCH p \= (o:Station {station\_id: $origin\_id})  
         \-\[:CONNECTS\_TO\*1..10\]-\>  
         (d:Station {station\_id: $destination\_id})  
WHERE NONE(n IN nodes(p) WHERE n.station\_id \= $avoid\_station\_id)  
RETURN \[n IN nodes(p) | {station\_id: n.station\_id, name: n.name}\] AS route,  
      reduce(t=0, r IN relationships(p) | t \+ r.travel\_time\_min) AS total\_time  
ORDER BY total\_time  
LIMIT $max\_routes  
\`\`\`

**\---**

**\#\#\#\#** **函數 5：\`query\_interchange\_path()\` — 跨網路換乘路線**

**\*\*輸入\*\***：\`origin\_id\`（地鐵或國鐵）, \`destination\_id\`（另一網路）

**\*\*輸出格式\*\***：  
\`\`\`python  
{  
   "found": True,  
   "stations": \[  
       {"station\_id": "MS01", "name": "Central Square"},  
       {"station\_id": "NR01", "name": "Central Station", "interchange": True},  
       ...  
   \],  
   "interchanges": \[{"from": "MS01", "to": "NR01", "transfer\_time\_min": 3}\],  
   "total\_time\_min": 25  
}  
\`\`\`

**\*\*Cypher\*\***：  
\`\`\`cypher  
MATCH p \= (o:Station {station\_id: $origin\_id})  
         \-\[:CONNECTS\_TO|INTERCHANGE\_WITH\*1..20\]-\>  
         (d:Station {station\_id: $destination\_id})  
WHERE any(r IN relationships(p) WHERE type(r) \= 'INTERCHANGE\_WITH')  
RETURN nodes(p), relationships(p)  
ORDER BY length(nodes(p))  
LIMIT 1  
\`\`\`

**\*\*實現重點\*\***：  
\- 混合使用 \`CONNECTS\_TO\` 和 \`INTERCHANGE\_WITH\`  
\- 標記換乘點（\`"interchange": True\`）  
\- 換乘時間固定估算 3 分鐘

\`\`\`python  
from sentence\_transformers import CrossEncoder

\_MODEL \= None  *\# lazy load*

def \_get\_model() \-\> CrossEncoder:  
   *global* \_MODEL  
   if \_MODEL is None:  
       \_MODEL \= CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")  
   return \_MODEL

def rerank(query\_text: str, results: list\[dict\], top\_k: int \= 5) \-\> list\[dict\]:  
   *"""*  
   *對初步搜尋結果重排序。*  
   *Args:*  
       *query\_text: 使用者原始問題（文字，非 embedding）*  
       *results:    query\_policy\_vector\_search 回傳的 list\[dict\]*  
       *top\_k:      最終回傳數量*  
   *Returns:*  
       *重排序後的 Top-k 結果*  
   *"""*  
   if not results:  
       return \[\]  
   model \= \_get\_model()  
   pairs \= \[(query\_text, r\["content"\]) for r in results\]  
   scores \= model.predict(pairs)  
   ranked \= sorted(zip(scores, results), key\=lambda x: x\[0\], reverse\=True)  
   return \[r for \_, r in ranked\[:top\_k\]\]  
\`\`\`

from databases.relational.queries import query\_policy\_vector\_search  
from skeleton.reranker import rerank

def search\_with\_rerank(  
   embedding: list\[float\],  
   query\_text: str,  
   category: str | None \= None,  
   top\_k: int \= 5,  
) \-\> list\[dict\]:  
   *"""*  
   *向量搜尋 \+ reranking 的完整流程。*  
   *Args:*  
       *embedding:  llm.embed(query\_text) 的結果*  
       *query\_text: 使用者原始問題（給 reranker 用）*  
       *category:   可選，限制搜尋的文件類別*  
       *top\_k:      最終回傳數量*  
   *Returns:*  
       *重排序後的 Top-k 政策文件*  
   *"""*  
   *\# 先拉 Top 20*  
   results \= query\_policy\_vector\_search(embedding, top\_k\=20)

   *\# Python 端 category 過濾（不動 SQL）*  
   if category:  
       results \= \[r for r in results if r.get("category") \== category\]

   *\# Reranking*  
   return rerank(query\_text, results, top\_k\=top\_k)
